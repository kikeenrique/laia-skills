# ComponentName (reference template)

This is a placeholder reference page. When authoring a skill, replace `componentname.md`
with one file per component/system you document (e.g. `reality-view.md`, `model3d.md`),
and link to it from the skill's `SKILL.md` component table.

Each reference page should cover:

- **What it is** — the API/type and its role.
- **When to use it** — the specific scenario that selects this component over alternatives.
- **How to use it** — a minimal, correct code example.
- **Gotchas** — platform constraints, lifecycle, and common mistakes.
